# ICT Time & Price Theory

## Core Principle
Time and price are fractal. The same patterns repeat across all timeframes - yearly, monthly, weekly, daily, sessions, and micro.

---

## Quarterly Theory (Jevaunie Daye)

Time must be divided into **quarters** for proper market cycle interpretation.

### Time Fractal Divisions

| Timeframe | Q1 | Q2 | Q3 | Q4 |
|-----------|----|----|----|----|
| **Yearly** | Jan-Mar | Apr-Jun | Jul-Sep | Oct-Dec |
| **Monthly** | Week 1 | Week 2 | Week 3 | Week 4 |
| **Weekly** | Monday | Tuesday | Wednesday | Thursday (Fri separate) |
| **Daily** | Asia (18:00-00:00) | London (00:00-06:00) | NY AM (06:00-12:00) | NY PM (12:00-18:00) |
| **Session** | 90min block 1 | 90min block 2 | 90min block 3 | 90min block 4 |
| **Micro** | 22.5min | 22.5min | 22.5min | 22.5min |

### Quarter Functions

- **Q1**: Accumulation / Range / Barometer for forecasting
- **Q2**: Manipulation / Judas Swing / False move
- **Q3**: Distribution / Expansion / True move
- **Q4**: Continuation or Reversal / Rebalancing

### True Open
The **True Open** marks the START of Q2 in every cycle:
- Daily True Open: 00:00 EST (London Open)
- Session True Open: Start of 2nd 90-min block
- Weekly True Open: Tuesday Open

**Trading Rule**: Buy BELOW True Open (bullish), Sell ABOVE True Open (bearish)

## Power of 3 (AMD)

Every price delivery follows: **Accumulation → Manipulation → Distribution**

### Weekly AMD Profile

| Day | Phase | Expected Action |
|-----|-------|-----------------|
| Monday | Accumulation | Range, establish weekly open |
| Tuesday | Manipulation | Judas swing, false breakout, stop hunt |
| Wednesday | Distribution | True expansion begins |
| Thursday | Distribution | Continuation of move |
| Friday | X (Seek & Destroy) | Reversal risk, reduced participation |

### Daily AMD Profile

| Time (EST) | Phase | Expected Action |
|------------|-------|-----------------|
| 18:00-00:00 | Accumulation | Asia range forms |
| 00:00-02:00 | Manipulation | Judas swing (London open) |
| 02:00-10:00 | Distribution | True daily move |
| 10:00-15:00 | X | NY PM rebalancing |

## IPDA (Interbank Price Delivery Algorithm)

Price is NOT random. It follows algorithmic rules targeting:
1. **Imbalances** (Fair Value Gaps)
2. **Liquidity** (Stop hunts)

### IPDA Data Ranges (Lookback)

| Range | Usage |
|-------|-------|
| 20-day | Short-term liquidity levels |
| 40-day | Medium-term swing points |
| 60-day | Major liquidity pools |

### IPDA Daily Targets
The algorithm seeks to fill **4 numbers daily**:
1. Previous day high/low
2. Current session liquidity
3. Fair Value Gaps
4. Equilibrium (50% of range)

### IPDA True Day
- **Starts**: 00:00 EST (midnight)
- **Ends**: 15:00 EST (3pm)
- After 15:00: Next day's accumulation begins

## Quarterly Shifts

Major market direction changes every **3-4 months**:

| Quarter | Typical Behavior |
|---------|------------------|
| Q1 (Jan-Mar) | Trend establishment |
| Q2 (Apr-Jun) | Continuation or reversal |
| Q3 (Jul-Sep) | Summer consolidation |
| Q4 (Oct-Dec) | Year-end distribution |

**Key dates**: Watch for reversals around quarter transitions (Mar 21, Jun 21, Sep 21, Dec 21)
